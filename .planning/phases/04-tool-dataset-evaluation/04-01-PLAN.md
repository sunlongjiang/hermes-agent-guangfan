**Three nested DSPy Signature classes** (inner classes of ToolDatasetBuilder):

1. `class AnalyzeToolSimilarity(dspy.Signature)`:
   - Input: `tool_summaries: str` (all tools as "- name: description" list)
   - Output: `confuser_pairs: str` (JSON array of `{"tools": ["tool_a", "tool_b"], "overlap": "description of overlap"}`)

2. `class GenerateToolTasks(dspy.Signature)`:
   - Input: `tool_name: str`, `tool_description: str`, `all_tools: str`, `difficulty: str`, `num_tasks: int`
   - Output: `tasks: str` (JSON array of `{"task_description": ..., "correct_params": {...}, "confuser_tools": [...]}`)

3. `class GenerateConfuserTasks(dspy.Signature)`:
   - Input: `tool_a_name: str`, `tool_a_description: str`, `tool_b_name: str`, `tool_b_description: str`, `overlap_description: str`, `num_tasks: int`
   - Output: `tasks: str` (JSON array of `{"task_description": ..., "correct_tool": ..., "correct_params": {...}, "reason": ...}`)

**Constructor**: `__init__(self, config: EvolutionConfig)` -- stores config, creates 3 `dspy.ChainOfThought` instances.

**generate() method** -- two-step process:
1. Call `AnalyzeToolSimilarity` with all tool summaries to get confuser pairs
2. For each tool, generate baseline examples (per D-09: 1 easy + 1 medium + 1 hard = 3 minimum per tool)
3. For each confuser pair, generate 5-10 confuser tasks (per D-06), all marked difficulty="hard" (per D-07)
4. Validate generated tool names against actual tool list (strip().lower() matching per Pitfall 1), discard unmatched
5. Check coverage: any tool with <3 examples gets supplementary generation
6. Adjust difficulty distribution toward 30/40/30 (per D-08) by controlling generation counts
7. Shuffle all examples, split 50/25/25 (per D-05), return ToolSelectionDataset

**JSON parsing**: Use Phase 1 two-stage strategy -- `json.loads()` first, regex `re.search(r'\[.*\]', text, re.DOTALL)` fallback (per Pitfall 4).

**Helper methods**:
- `_validate_tool_name(name: str, valid_names: list[str]) -> Optional[str]`: strip().lower() matching against valid tool names, returns matched name or None
- `_ensure_coverage(examples: list[ToolSelectionExample], tool_names: list[str]) -> list[str]`: returns list of tool names with <3 examples
- `_parse_json_array(text: str) -> list[dict]`: two-stage JSON parsing

Use `dspy.context(lm=dspy.LM(self.config.judge_model))` for all LLM calls (same as SyntheticDatasetBuilder).

Rich console output: progress messages via `console.print()` for each step (similarity analysis, per-tool generation, confuser generation, coverage check).

For tests: mock `dspy.ChainOfThought` to return canned JSON responses. Test the validation/coverage/parsing logic with real data, mock only LLM calls.
