    **创建 `evolution/tools/tool_constraints.py`:**

    1. 模块级 docstring 说明用途：工具描述的事实准确性验证。
    2. 导入：`dspy`, `from evolution.core.config import EvolutionConfig`, `from evolution.core.constraints import ConstraintResult`。
    3. 创建 `_parse_bool(value) -> bool` 辅助函数（per D-02 的 Pass/Fail 二值需求）：
       - `isinstance(value, bool)` 直接返回
       - `str(value).strip().lower()` 与 `("true", "yes", "1")` 比对
       - 默认返回 False
    4. 创建 `ToolFactualChecker` 类（per D-01, D-02, D-03）：
       - 内部类 `FactualCheckSignature(dspy.Signature)`，docstring 说明"对比原始和进化描述，检测虚假能力声明"
         - InputField: `tool_name: str`, `original_description: str`, `evolved_description: str`
         - OutputField: `has_false_claims: bool` (desc="True if evolved claims capabilities NOT in original"), `explanation: str`
       - `__init__(self, config: EvolutionConfig)`: 存 config，创建 `self.checker = dspy.ChainOfThought(self.FactualCheckSignature)`
       - `check(self, tool_name: str, original_description: str, evolved_description: str) -> ConstraintResult`:
         - `lm = dspy.LM(self.config.eval_model)`
         - `with dspy.context(lm=lm):` 调用 self.checker
         - 用 `_parse_bool(result.has_false_claims)` 解析结果
         - 返回 `ConstraintResult(passed=not has_false, constraint_name="factual_accuracy", message=..., details=str(result.explanation))`
       - `check_all(self, original_tools: list, evolved_tools: list) -> list[ConstraintResult]`:
         - 构建 `original_map = {t.name: t for t in original_tools}`
         - 遍历 evolved_tools，从 original_map 找到对应原始工具
         - 对每个匹配的对调用 `self.check(evolved.name, original.description, evolved.description)`
         - 返回结果列表
