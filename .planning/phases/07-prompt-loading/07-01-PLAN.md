       - `_format_paren_concat(var_name: str, text: str, indent: int = 4) -> str`: Format text as parenthesized concat assignment. Output format:
         ```
         VAR_NAME = (
             "line1 "
             "line2"
         )
         ```
         Split text into ~70 char lines. Escape internal double quotes. Each line except last gets trailing space before closing quote.

       - `_format_dict_value_paren_concat(text: str, indent: int = 8) -> str`: Format text as a parenthesized concat value suitable for inside a dict. Output format (indented at 8 spaces for dict nesting):
         ```

                 "line2"
         ```
         This replaces only the string content lines within a dict value -- the `"key": (` prefix and `)` suffix are preserved from the original file.

       CRITICAL per Pitfall 4: When writing back multiple sections, process from bottom of file upward (highest line_range first) so earlier sections' line numbers remain valid. Document this in the function docstring. The `write_back_section` function itself handles single sections -- the caller must sort sections by line_range descending for batch writes.

       CRITICAL per Pitfall 5: After formatting, the replacement must produce valid Python. Use `repr()` or manual escaping to handle quotes within evolved text.

| hermes-agent filesystem -> prompt_loader | Read-only access to prompt_builder.py; file content is trusted (developer-authored Python) |
| evolved text -> write-back | Evolved text from LLM inserted into Python source file as string literals |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | write_back_section | mitigate | py_compile validation after every write-back ensures no syntax corruption; string escaping via repr() or manual quote escaping prevents code injection |
| T-07-02 | Information Disclosure | extract_prompt_sections | accept | Extracts text from local files only; no network access; hermes-agent is developer's own repo |
| T-07-03 | Tampering | _format_paren_concat | mitigate | Evolved text is placed inside string literals only -- cannot become executable Python code; line-level replacement preserves all non-target code |
</threat_model>
